# 데이터 프로파일링 상세 절차

## A-1. 데이터 소스 판별

| 입력 유형 | 처리 |
|----------|------|
| 파일 경로 (.csv, .xlsx, .txt, .dat, .parquet) | `pd.read_csv/read_excel/read_parquet` |
| DB 테이블명 | CLAUDE.md DB 접속정보 → `pd.read_sql` |
| DataFrame 변수명 | 현재 컨텍스트 참조 |
| SQL 쿼리 | 쿼리 실행 후 프로파일링 |
| 디렉토리 경로 | 하위 파일 목록 → 일괄 프로파일링 |

### 로드 전 확인
- **파일**: 존재, 크기, 인코딩 순차 시도 (utf-8 → cp949 → euc-kr → latin-1)
- **DB**: 연결 가능 여부 (실패 시 VPN 확인 안내)
- **대용량 판단**: 1GB+ 또는 1,000만건+ → 샘플링 제안

## A-2. 프로파일링 수행

### 기본 정보
| 항목 | 수집 방법 |
|------|----------|
| 행 수 | `len(df)` |
| 열 수 | `len(df.columns)` |
| 메모리 | `df.memory_usage(deep=True).sum()` → MB |
| 중복 행 | `df.duplicated().sum()` + 비율 |
| 데이터 타입 분포 | `df.dtypes.value_counts()` |
| 전체 결측률 | `df.isna().sum().sum() / df.size` |

### 컬럼별 분석

**수치형** (int, float):
- 시각화 먼저 (Tukey): 히스토그램/분포도
- 기초통계: count, mean, std, min, Q1, median, Q3, max
- 이상치: IQR (Q1-1.5xIQR ~ Q3+1.5xIQR 범위 밖)
- 0값 비율, 음수값 비율, 왜도, 첨도

**범주형** (object, category):
- 고유값 수, 최빈값+빈도 (상위 10개), 값 길이 분포, 빈 문자열 건수

**날짜형** (datetime):
- 범위, 결측 구간, 주기 패턴, 미래 날짜 여부

### 품질 검사

| 점검 항목 | 기준 | 심각도 |
|----------|------|--------|
| 결측률 30%+ | 활용 가치 의문 | Warning |
| 결측률 80%+ | 컬럼 제거 검토 | Critical |
| 상수 컬럼 | 정보 없음 | Warning |
| 전체 NULL | 즉시 제거 | Critical |
| 혼합 타입 | 숫자+문자 혼재 | Critical |
| 이상치 5%+ | 품질 의심 | Warning |

### 관계 분석
- 피어슨 상관계수, |r| > 0.8 하이라이트
- 키 후보: 고유값 비율 = 1.0
- 참조 무결성, 논리 규칙

## A-비교. 데이터 비교

| 비교 축 | 내용 |
|---------|------|
| 스키마 | 컬럼 수, 이름, 타입, 순서 차이 |
| 건수 | 전체 행 수, 키 기준 매칭, 중복 |
| 값 | 셀 단위 불일치 (키 컬럼 지정 시) |
| 통계 | 수치형 기초통계, 범주형 빈도 분포 |
| 분포 | KS 검정, 카이제곱 검정 |

### 값 비교 규칙
- 수치형: epsilon = 1e-6
- 문자열: 공백/대소문자 차이 구분
- NULL끼리는 "일치"
- 키 미지정 → 고유값 비율 = 1.0 컬럼 자동 추론

## D. 자동 EDA 파이프라인

1. 데이터 로드 + 기본 프로파일링
2. 수치형: 분포 + 이상치 시각화
3. 범주형: 빈도 + 상위 N개
4. 날짜형: 시계열 트렌드
5. 상관관계 히트맵
6. 결측 패턴 분석
7. 타겟 변수 관계 분석
8. 핵심 인사이트 요약 + modeling_hints 섹션

### 이상치 분석
- IQR 방식 + Z-score (|z| > 3)
- 이상치 제거 전 '왜 이상한가' 먼저 탐색 (Tukey 잔차 원칙)
- 제거 전후 통계 비교
