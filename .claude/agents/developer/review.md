# 코드 리뷰 상세

## B-1. 대상 파악
| 입력 | 동작 |
|------|------|
| 파일 경로 | 해당 파일 전체 읽기 |
| 폴더 경로 | 하위 .py, .r, .sql 전체 |
| 인자 없음 | git diff 변경 파일 대상 |
| PR 번호 | gh pr diff 변경사항 추출 |

필수: 대상 코드 전체 읽기 + import 라이브러리 파악 + 호출 모듈 읽기

## B-2. 4차원 리뷰

### 1. 정확성
- 로직 오류, 경계값, off-by-one, 타입 불일치
- 반환값 일관성, 데이터 변환 (인코딩/날짜/소수점)
- Karpathy Recipe 준수: 단순 버전에서 시작? 불필요 의존성?

### 2. 보안
- SQL 인젝션 (f-string → 바인드 변수)
- 크레덴셜 노출 (하드코딩 → .env)
- 경로 조작, 민감정보 로깅

### 3. 성능
- 벡터화 미활용 (for문 → vectorized)
- N+1 쿼리, 메모리 폭발, 리소스 누수
- 불필요 복사, 중복 계산

### 4. 유지보수성
- 중복 코드, 매직넘버, 하드코딩 경로
- 함수 50줄 초과, 네이밍 컨벤션

## B-3. 심각도별 정책
| 심각도 | 정책 |
|--------|------|
| Critical | 사용자 확인 후 즉시 수정 |
| Warning | 수정 여부 확인 |
| Suggestion | 제안만 |
